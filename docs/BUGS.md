# Bugs — Agent and User Reports

When an agent or user hits a failure, we document it here. Root cause and "agent-intuitive" implications matter: **is the CLI intuitive enough for the LLM?** See [VISION.md](./VISION.md) (agent-first, agent-intuitive interfaces).

| [BUG-017](bugs/BUG-017-semantic-recall-gap-intent-queries.md) | Semantic Recall Gap for Intent-Based Queries — Agent-Reported | Bakeoff #2: zmail missed Marcio Nunes (Harmonee AI, Mar 3) despite data being in sync range. "What emails did I get today?" scores 0.40–0.60 despite listing the right emails. Workaround: `minScore: 0.3` on affected test. Fix: add fixture context to judge prompt or use deterministic checks. |
| [BUG-024](bugs/BUG-024-inbox-scan-over-filters-misses-important-mail.md) | `zmail inbox` Over-Filters — Misses Important Mail | Inbox LLM acts as final curator (narrow `newMail`) instead of a coarse junk stripper. Fix: prompt + model/caps/modes; optional inclusive mode or return full candidates. |
